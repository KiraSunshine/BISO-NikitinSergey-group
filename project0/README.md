### 3. Ответы на контрольные вопросы


#### 1. Для чего нужна команда `git init, git clone`?


Команда git init создает новый пустой репозиторий в текущей папке на компьютере. Команда git clone копирует существующий репозиторий с сервера на компьютер. После git clone у меня появилась папка practice0 с проектом.


#### 2. Что такое "коммит"?


Коммит — это как фотография состояния проекта в определенный момент времени. В него входит: список измененных файлов, автор коммита, дата и время, а также мой комментарий (что было сделано). Git хранит всю историю коммитов, чтобы можно было вернуться к любому из них.


#### 3. Чем отличаются файлы `.puml` от `.png` в контексте вашей работы?


Файл .puml — это исходный код диаграммы, написанный человеком. Его можно читать и изменять. Файл .png — это готовая картинка, которую понимает человек, но компьютер не может её легко изменить. В Git лучше хранить .puml, так как он занимает мало места, и видно историю его изменений. Картинку всегда можно сгенерировать заново из кода.


#### 4. Объясните, как в коде Go реализуется связь "класс User использует интерфейс Notifier", показанная на диаграмме.


В коде интерфейс Notifier описывает правило — у него должна быть функция Send. Структуры EmailNotifier и SmsNotifier выполняют это правило (реализуют интерфейс). Структура UserNikitinSergey имеет поле notifier с типом Notifier (интерфейс). Когда мы создаем пользователя в функции NewUserNikitinSergey, мы передаем туда конкретный способ уведомления — или EmailNotifier, или SmsNotifier. Преимущество: если мы захотим добавить новый способ (TelegramNotifier), нам не нужно переписывать код пользователя.
